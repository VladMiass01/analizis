Задачи кейса по маркетплейсу Amazon
В этом кейсе мы будем работать с данными о продажах в США за 2020 год на платформе Amazon.
В этом кейсе присутствуют следующие поля: дата заказа, ID записи (который не имеет значения для дальнейшего анализа), ID заказа, тип доставки, ID клиента, сегмент клиента, страна (в данном случае только США), город, штат, почтовый индекс, регион, ID продукта, категория товара, подкатегория товара, название товара, общая сумма продаж, количество проданных единиц, скидка и прибыль (которая нам не понадобится).
Первоначальные действия не отличаются от алгоритма предыдущих кейсов: приведи столбцы к стилю camel case, преобразуй типы данных (например, даты к формату datetime), найди и удали/замени дубли и пропуски.
В основной части исследовательского анализа выполни следующие действия:
•	узнай, сколько уникальных заказов и сколько уникальных клиентов содержится в таблице;
•	изучи локацию пользователей. Рекомендуем построить три столбчатые диаграммы по регионам, штатам и городам, отсортировать их и добавить текстовый комментарий, который объяснит, где больше заказов, а где меньше;
•	изучи динамику количества заказов по дате заказа. Сгруппируй данные по дате заказа, посчитай количество заказов и визуализируй результат - скорее всего, при этом нужно будет усечь данные до месяца. Отметь падение (либо рост), после чего отрази это в текстовой ячейке;
•	изучи динамику продаж по сумме (поле Sales);
•	изучи распределение продаж по категориям заказов. Здесь нужно создать сводную таблицу, группируя данные по категориям и подкатегориям, суммируя продажи по полю Sales. Визуализируй результат на двух графиках в формате столбчатой диаграммы: один для суммарных продаж по категориям, другой под подкатегориям. Оцени в текстовом выводе, какие категории и подкатегории лидируют, а какие находятся в конце списка;
•	проанализируй распределение продаж по городам, штатам и регионам, а также найди закономерности;
•	найди самые популярные товары. Сгруппируй данные по полю Product Name, посчитай количество заказов и составь список топ-10 или топ-100 товаров, визуализируя результаты на столбчатой диаграмме;
•	изучи распределение категорий внутри города или штата. Здесь можно сгруппировать данные по городу и по категории, посчитать количество заказов и визуализировать полученные результаты. Возможно, ты увидишь, что для каждого города есть свой паттерн - например, в Калифорнии заказывают больше фурнитуры, а жители Лос-Анджелеса больше покупают товаров из категории "Технологии". Можно углубиться до подкатегорий. Установленные закономерности опиши в текстовой ячейке;
•	проанализируй динамику продаж внутри категорий и подкатегорий. Отметь пики и спады продаж внутри каждой категории и внутри каждой подкатегории, сравни с общим графиком динамики продаж и сформируй выводы.
В заключение опиши все выявленные закономерности по данному кейсу.
